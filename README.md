# Tagify v8.1 — 图片标签管理系统

基于深度学习的图片标签管理系统，支持自动标注、多维度搜索和可视化浏览。使用 **WD ViT Tagger v3** 模型，识别精准、细节丰富。

## 模型升级亮点

| 特性 | 旧模型 (ResNet) | 新模型 (WD ViT v3) |
|:---|:---:|:---:|
| **置信度分布** | 虚高 (90-100%) | 真实梯度分布 (5-99%) |
| **标签准确性** | 偶现错误标签 | 精准识别 |
| **细节标签** | 较少 | 5-50%区间包含丰富细节 |
| **角色识别** | 基础 | 精准识别东方Project角色 |
| **场景理解** | 一般 | 优秀（可识别"拥抱"、"睡觉"等场景） |

## 运行准备

### 1. 文件夹结构

```
项目目录/
├── ntest.py              # 主程序
├── app_config.json        # 应用配置文件（修改此文件调整参数）
├── app_config.md          # 配置说明文档
├── model.safetensors      # 模型权重
├── config.json            # 模型架构配置
├── selected_tags.csv      # 标签文件
├── image_tags.db          # SQLite 数据库（首次运行自动创建）
├── input_image/           # 存放待处理的图片
└── gallery/               # 处理后的图片存档
```

### 2. 模型文件下载

前往 [HuggingFace 模型库](https://huggingface.co/SmilingWolf/wd-vit-tagger-v3) 下载以下文件：

- `model.safetensors` — 模型权重
- `config.json` — 模型配置文件
- `selected_tags.csv` — 标签文件

### 3. 配置参数

**不再需要修改代码！** 所有可调参数都在 `app_config.json` 中，修改后重启即可生效。详见 `app_config.md`。

常用调整示例：

| 需求 | 修改 `app_config.json` 中的键 |
|------|------------------------------|
| 标签太多/太少 | `model.process_threshold`（调大减少，调小增加） |
| 缩略图大小 | `ui.thumbnail_size` |
| 每页图片数 | `ui.page_size` |
| 窗口尺寸 | `ui.window_size` |
| 收藏标签名 | `behavior.favorite_tag` |
| 默认排序 | `behavior.default_sort` / `default_order` |

## 使用指南

### 1. 图片处理

- 将图片放入 `input_image/` 文件夹
- 点击 **"开始批量处理"** 按钮
- 程序自动标注标签，存入 `image_tags.db`，并将图片移动到 `gallery/`

### 2. 标签搜索

- 在左侧搜索框输入关键词，点击 **"搜索"**
- 下方显示匹配的标签及包含该标签的图片数量
- 点击标签即可筛选

### 3. 图片浏览

- 点击 **"显示图库"** 查看所有图片
- 点击标签后中间面板显示含此标签的缩略图
- 拖动面板边框可调整三栏宽度，中间面板自动适配每行列数
- **单击**缩略图：右侧面板显示详细信息和标签
- **双击**缩略图：查看原图（同一张图不会重复弹窗）
- **右键**缩略图/原图：复制图片、收藏/取消收藏、删除图片
- 收藏功能自动给图片添加指定标签（默认 `collect`）
- 底部有分页按钮，顶部支持按名称/大小/时间排序

### 4. 右侧标签详情

- 上方显示图片基本信息（名称、大小、处理时间）
- 标签分为**主要标签**（置信度 > 阈值）和**详细标签**（低置信度区间）
- 勾选 **"显示更多标签"** 可查看低置信度标签
- 右键标签可复制到剪贴板

### 5. 数据完整性检查

- 建议定期点击 **"检查数据完整性"** 按钮
- 会对比数据库记录与实际文件，报告不一致项

- 首次运行自动创建 `image_tags.db` 数据库
- 建议定期点击 **数据完整性检查** 按钮
- GPU 用户可获得更快的处理速度
- 5-50% 的标签同样准确，可在右侧勾选显示
- 如有大量图片，建议分批处理
- 修改 `app_config.json` 后重启程序生效

3. 通过 `app_config.json` 调整参数，不再需要修改代码
